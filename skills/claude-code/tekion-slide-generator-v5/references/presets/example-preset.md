# Design Specifications

## 配色パレット

### ブランドカラー（Primary）
- Primary: #EA5514 - ブランドのメインカラー。見出し装飾、アイコン、矢印、ボタン、強調に使用
- Primary Light: #FEF0E8 - オレンジ系の薄い背景。ハイライトエリア、注目ボックスの背景
- Primary Dark: #B8400F - 高コントラストが必要な場面。白テキストの背景色、中扉グラデーション暗部

### アクセントカラー（データ可視化・差別化用）
- Accent Teal: #0D7C72 - 図表の第3系列、差別化アクセント
- Accent Gold: #C4880D - 図表の第4系列、ハイライト、注意表示

### セマンティックカラー（状態表示用）
- Success: #1A7A3D - 完了、正解、推奨（フォレストグリーン）
- Warning: #C4880D - 注意、要確認（Accent Goldと共用）
- Error: #C52B2B - エラー、禁止、重要警告（クリムゾンレッド）
- Info: #2D6CB4 - 参考情報、ヒント、補足（インフォブルー）

### グレースケール（5段階）
- Gray-900: #1A1A1A - 見出しテキスト
- Gray-700: #4A4A4A - 本文テキスト
- Gray-400: #9CA3AF - キャプション、補足テキスト
- Gray-200: #E5E7EB - 罫線、セパレーター
- Gray-100: #F3F4F6 - カード背景、コンテナ背景
- ベース: #FFFFFF - スライド背景

### 配色ルール
- 1スライドあたりの使用色は最大3色（グレースケール除く）
- 60-30-10ルール: 60%ニュートラル（白/グレー）、30%Primary Lightまたは背景色、10%アクセント
- セマンティックカラーは状態表示の目的でのみ使用し、装飾には使わない
- データ可視化ではPrimary → Accent Teal → Accent Goldの順で系列に割り当て

## フォント

- 見出し: 太字の丸みのあるゴシック系書体
- 本文: 丸みのあるゴシック系書体

## 技術仕様

- アスペクト比: 16:9
- 解像度: 2752x1536px (2K)
- フッター: なし（ロゴは --logo オプションで制御。テキストフッターは入れない）
- 余白: 全辺に5%程度

## レイアウトパターン

### Pattern A: スプリット(左右対比)

用途: 比較(Before/After)、対比、2つの概念
構造: 左50%テキスト + 右50%ビジュアル
視覚要素: イラスト、写真、アクセントカラーの境界線

---

### Pattern B: プロセス・フロー

用途: 手順、時系列、サイクル
構造: 左から右または上から下の流れ、角丸ボックス + 太い矢印
視覚要素: ステップ番号、フローチャート、矢印にアクセントカラー

---

### Pattern C: グリッド・コンテナ

用途: 3つ以上のポイント、リスト、比較
構造: 等分割グリッド(2x2, 3x1など)、各セルはアイコン付きカード
視覚要素: Primaryカラーのアイコン、Gray-100またはPrimary Lightのカード背景

---

### Pattern D: センター・フォーカス

用途: 重要な数字、引用、結論
構造: 画面中央に巨大な要素、背景に幾何学模様
視覚要素: 大きな数字/フレーズ、アクセントカラーのサークル

---

### Pattern E: 構造化ダイアグラム

用途: 階層構造、体系図、組織図
構造: ピラミッド型、同心円型、ツリー型
視覚要素: 図形(Gray-100/Primary/Accent Teal)、関係線、矢印

---

### Pattern F: 大胆なイラスト中心

用途: 抽象的な概念、感情、ストーリー、メタファー
構造: 画面の60-70%を占める大きなイラスト + 最小限のテキスト
視覚要素: アイソメトリック図、メタファーを表すイラスト、象徴的なビジュアル

---

### Pattern G: 全画面ビジュアル + オーバーレイテキスト

用途: インパクト重視、感情的訴求、導入スライド
構造: 全画面の写真/イラスト背景、その上にテキストをオーバーレイ
視覚要素: 半透明ボックス、白抜き文字

---

### Pattern H: アイコンマトリクス

用途: 多数の小項目、チェックリスト、機能一覧
構造: 4x3または3x4のアイコングリッド、各セルにアイコン+1行テキスト
視覚要素: アクセントカラーのシンプルなアイコン、最小限のラベル

---

### Pattern I: タイムライン/ロードマップ

用途: 歴史、計画、進化、段階的プロセス
構造: 横軸または縦軸の時間線、マイルストーンをポイントで表示
視覚要素: アクセントカラーの時間線、年号/日付、マイルストーンのドット

---

### Pattern J: データビジュアライゼーション

用途: 統計、比較、割合、トレンド
構造: グラフ、チャート、インフォグラフィック
視覚要素: 棒グラフ、円グラフ、折れ線グラフにアクセントカラー

---

### Pattern K: コンセプトイラスト全面

用途: 複雑な概念の視覚化、システム全体像、関係性の表現
構造: 画面全体を使った大きな概念図、テキストは図の中に埋め込み
視覚要素: 接続線、ノード、レイヤー構造、アノテーション

---

### Pattern L: 中扉（セクション区切り）

用途: 章の開始、大きなトピック転換、セクション区切り
構造: 画面中央にタイトルとサブタイトルを配置、背景に幾何学模様または抽象的なビジュアル
視覚要素:
- 大きなタイトル（画面中央、太字、白テキスト）
- サブタイトル（タイトル下、やや小さめ、白テキスト）
- 背景パターン:
  - オレンジグラデーション: Primary(#EA5514) → Primary Dark(#B8400F)
- 最小限の装飾（アイコン、線、シェイプなど）
- テキストは最小限、視覚的インパクト重視

---

### Pattern M: 講座タイトル（表紙）

用途: プレゼンテーション冒頭、講座・研修の表紙
構造: 3層構造
- 上部: 講座名を大きく中央配置
- 中央: 組織名を中サイズで表示
- 下部: 回次・テーマをやや小さめに表示
視覚要素:
- 背景: 白ベース、Primary(#EA5514)の幾何学ライン + Gray-900(#1A1A1A)のテキスト
- ロゴエリア（あれば上部に配置）
- 幾何学模様やグラデーション（控えめ）

## 図解パターンガイド

コンテンツに応じて以下の図解タイプを選択し、対応するレイアウトパターンの構造内で描画する。

### 階層・構造系（Pattern E ベース）
- **ツリー図**: 上→下の親子接続線、階層が深くなるほど細分化。組織図、カテゴリ分類に最適
- **ピラミッド図**: 底辺=基礎/土台、頂点=目標/上位概念。段階的階層を三角形で表現
- **放射図**: 中心概念から放射状に広がる要素。影響範囲、ステークホルダーマップに使用
- **階層・レイヤー**: 3D積層効果で基盤→応用の関係を表現。技術スタック、インフラ構成に最適
- **TAM-SAM-SOM図**: 同心円3層（外=TAM、中=SAM、内=SOM）。市場規模分析に使用

### プロセス系（Pattern B ベース）
- **フロー図（横型）**: 左→右の流れ、太い矢印+ステップ番号。業務フロー、ユーザージャーニーに最適
- **フロー図（縦型）**: 上→下の流れ、各ステップに詳細記述スペース。意思決定フロー、手順書に使用
- **サイクル図**: 円形配置+循環矢印、開始/終了なし。PDCA、改善サイクルの表現に最適
- **階段・ステップ図**: 階段状の段階的上昇。成長ステージ、習熟度レベルの表現に使用
- **ビフォーアフター**: Pattern A併用。左=Before、右=After、中央に変化の矢印。導入効果の対比に最適

### グルーピング系（Pattern C/H ベース）
- **グループ図（少数）**: Pattern C。3-5個の大きなカード、各カードにアイコン+詳細テキスト
- **グループ図（多数）**: Pattern H。6個以上のコンパクトカード、ロゴやアイコン中心の配置
- **パズル・ハニカム**: Pattern C。六角形タイルの隙間ない配置、中心と周辺の関係表現に使用
- **マトリクス**: Pattern C。2軸グリッド+四象限ラベル。競合分析、優先順位分類に最適
- **項目比較図**: Pattern A/C併用。横並び特徴比較、自社=有彩色/他社=無彩色のコントラスト

### 関係性・概念系（Pattern K/D ベース）
- **相関図**: Pattern K。ノード+接続線、線の太さで関係の強弱を表現。ビジネスモデル図に最適
- **ベン図**: Pattern K。重なる円、交差部分を強調色で表現。共通点・差異の可視化に使用
- **地図・マップ**: Pattern K。地理的/空間的な位置関係を表現。拠点展開、エリア分析に使用
- **トライアングル**: Pattern D。正三角形の3頂点=相互依存要素、中心に目的を配置

### 強調・数値系（Pattern D/J ベース）
- **数式**: Pattern D。画面中央に大きく数式形式で表現。成果の構成要素、価値の公式に使用
- **規模比較図**: Pattern D/J。円の大きさで規模差を表現、数値ラベル必須。市場規模、売上比較に使用

### 図解選択ガイドライン

1. **項目数で判断**: 3項目以下→Pattern D系、4-5項目→Pattern C系、6項目以上→Pattern H系
2. **方向性で判断**: 時系列・手順→フロー図、階層→ツリー/ピラミッド、対比→ビフォーアフター
3. **関係性で判断**: 因果関係→フロー図、相互依存→トライアングル/相関図、包含関係→ベン図
4. **強調対象で判断**: 数字→数式/規模比較、全体像→相関図/地図、データ→Pattern J系
