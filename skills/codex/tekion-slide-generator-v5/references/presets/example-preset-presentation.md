# Design Specifications — Presentation Mode

## 最重要原則: 1スライド1メッセージ

- 1枚のスライドで伝えるメッセージは**1つだけ**
- テキストは**最大3行**。それ以上は別スライドに分割する
- 箇条書きは**最大3項目**。4つ以上なら2枚に分ける
- 聴衆が3秒で「何の話か」わかるスライドだけが合格

## テキスト量の厳格な上限

- タイトル: 15文字以内
- サブタイトル/キーメッセージ: 25文字以内
- 本文テキスト合計: **50文字以内**（コードブロック除く）
- 表は最大3行x3列
- 余白を恐れない。**白い部分が多いほど良いスライド**

## ビジュアルファースト

- スライド面積の**60%以上をビジュアル**（図解・イラスト・アイコン・写真）が占める
- テキストだけのスライドは禁止。必ず視覚要素を入れる
- 比喩や概念は**メタファーイラスト**で表現する（例: 「指揮者」→オーケストラのイラスト）
- 技術概念は**図解・フロー・ダイアグラム**で表現する
- 数値データは**巨大な数字**で画面中央に配置（Pattern D）
- コード例は**最小限（3行以内）**、シンタックスハイライト風の装飾付き

## 配色パレット

### ブランドカラー（Primary）
- Primary: #EA5514 - ブランドのメインカラー。見出し装飾、アイコン、矢印、ボタン、強調に使用
- Primary Light: #FEF0E8 - オレンジ系の薄い背景。ハイライトエリア、注目ボックスの背景
- Primary Dark: #B8400F - 高コントラストが必要な場面。白テキストの背景色、中扉グラデーション暗部

### アクセントカラー（データ可視化・差別化用）
- Accent Teal: #0D7C72 - 図表の第3系列、差別化アクセント
- Accent Gold: #C4880D - 図表の第4系列、ハイライト、注意表示

### セマンティックカラー（状態表示用）
- Success: #1A7A3D - 完了、正解、推奨（フォレストグリーン）
- Warning: #C4880D - 注意、要確認（Accent Goldと共用）
- Error: #C52B2B - エラー、禁止、重要警告（クリムゾンレッド）
- Info: #2D6CB4 - 参考情報、ヒント、補足（インフォブルー）

### グレースケール（5段階）
- Gray-900: #1A1A1A - 見出しテキスト
- Gray-700: #4A4A4A - 本文テキスト
- Gray-400: #9CA3AF - キャプション、補足テキスト
- Gray-200: #E5E7EB - 罫線、セパレーター
- Gray-100: #F3F4F6 - カード背景、コンテナ背景
- ベース: #FFFFFF - スライド背景

### 配色ルール
- 1スライドあたりの使用色は最大3色（グレースケール除く）
- 60-30-10ルール: 60%ニュートラル（白/グレー）、30%Primary Lightまたは背景色、10%アクセント

## フォント

- 見出し: **超太字**の丸みのあるゴシック系書体。タイトルは画面の1/4を占めるサイズ
- 本文: 丸みのあるゴシック系書体。**24pt相当以上**（小さい文字は禁止）

## 技術仕様

- アスペクト比: 16:9
- 解像度: 2752x1536px (2K)
- フッター: なし（ロゴは --logo オプションで制御）
- 余白: 全辺に**8%以上**（ゆったりとした余白）

## レイアウトパターン

### Pattern A: スプリット(左右対比)
用途: 比較(Before/After)、対比、2つの概念
構造: 左50%テキスト + 右50%ビジュアル（大きなイラスト or アイコン）

### Pattern B: プロセス・フロー
用途: 手順、時系列、サイクル
構造: 大きなフロー図が画面の70%を占める。ステップ番号 + 太い矢印

### Pattern C: グリッド・コンテナ
用途: 3つのポイント（最大3つ）
構造: 3分割グリッド、各セルは大きなアイコン + 1行テキスト

### Pattern D: センター・フォーカス
用途: 重要な数字、引用、結論、キーメッセージ
構造: 画面中央に巨大なテキストまたは数字。背景に控えめな幾何学模様

### Pattern E: 構造化ダイアグラム
用途: 階層構造、体系図、レイヤー図
構造: 大きな図解が画面の80%を占める。ラベルは最小限

### Pattern F: 大胆なイラスト中心
用途: 抽象的な概念、メタファー、ストーリー
構造: 画面の70-80%を占める大きなイラスト + 1行のキャプション

### Pattern G: 全画面ビジュアル + オーバーレイテキスト
用途: インパクト重視、感情的訴求、導入スライド
構造: 全画面の写真/イラスト背景 + 白抜き大文字テキスト

### Pattern H: アイコンマトリクス
用途: 機能一覧、チェックリスト（最大6項目）
構造: 大きなアイコン + 1行ラベル

### Pattern I: タイムライン/ロードマップ
用途: 歴史、進化、段階的プロセス
構造: 横軸の大きな時間線。マイルストーンを3-4個のみ表示

### Pattern J: データビジュアライゼーション
用途: 統計、比較、トレンド
構造: 巨大な数字 or シンプルなグラフが画面の70%を占める

### Pattern K: コンセプトイラスト全面
用途: 複雑な概念の視覚化、システム全体像
構造: 画面全体を使った大きな概念図。テキストは図の中に埋め込み

### Pattern L: 中扉（セクション区切り）
用途: 章の開始、大きなトピック転換
構造: 画面中央に大きなタイトル + サブタイトル。オレンジグラデーション背景

### Pattern M: 講座タイトル（表紙）
用途: プレゼンテーション冒頭
構造: 講座名を超大きく中央配置。幾何学ライン装飾
